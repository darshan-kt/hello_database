# Project 3: Hospital AI Knowledge Assistant

## Professional RAG Development Roadmap

## Project Goal

Build an enterprise-grade Hospital AI Knowledge Assistant using
Retrieval-Augmented Generation (RAG).

The assistant should answer hospital-related questions by retrieving
information from an internal hospital knowledge base instead of
relying solely on an LLM's pretrained knowledge.

The system should be modular, production-ready, and designed following
software engineering best practices.

### Technology Stack

- Python
- PostgreSQL
- pgvector
- FastAPI
- LangChain (or LlamaIndex)
- OpenAI/Open-source Embedding Model
- OpenAI/Open-source LLM

## Engineering Mindset

Before writing any code, always answer these questions:

1. What business problem are we solving?
2. Who are the users?
3. What information will the AI retrieve?
4. Where is the information stored?
5. How should documents be indexed?
6. How should retrieval work?
7. What should happen if retrieval fails?
8. How should hallucinations be minimized?
9. How should the system scale?
10. How should responses be evaluated?

## Module 0 — Requirement Analysis

**Business Problem**

Hospitals contain thousands of documents. Examples:

- Clinical Guidelines
- SOP Documents
- Patient Education Material
- Drug Information
- Insurance Policies
- Medical Research
- Lab Reports
- Discharge Summaries

Doctors and staff waste time manually searching PDFs.

**Goal**

Allow users to ask natural language questions, e.g. *"What is the
treatment protocol for dengue?"* The AI should retrieve the correct
hospital document before answering.

**Deliverables**

- Business Requirements
- User Stories
- Functional Requirements
- Non-functional Requirements

---

## Module 1 — Domain Modeling

Identify business entities:

- Patients
- Doctors
- Departments
- Appointments
- Medicines
- Prescriptions
- Lab Reports
- Hospital Documents
- Document Chunks
- Embeddings
- Questions
- Answers

For every entity, define: purpose, attributes, relationships,
ownership, lifecycle.

---

## Module 2 — Database Design

Design relational tables. Suggested tables:

- patients
- doctors
- departments
- appointments
- medicines
- prescriptions
- lab_reports
- hospital_documents
- conversation_history
- users
- audit_logs

**Deliverables**: ER Diagram, Relationships, Normalization, Constraints,
Indexes.

---

## Module 3 — Vector Database Design

Enable pgvector. Create vector-specific tables, e.g.
`document_chunks`, `chunk_embeddings`, `metadata`.

Store: Document ID, Chunk ID, Chunk Text, Embedding Vector, Metadata
(Source, Department, Created Time).

**Deliverables**: Vector Schema, Embedding Strategy, Metadata Design.

---

## Module 4 — Document Ingestion Pipeline

Support: PDF, Word Documents, TXT, Markdown.

Pipeline: Upload → Text Extraction → Cleaning → Chunking → Embedding →
Store into PostgreSQL.

**Deliverables**: Document Loader, Chunking Service, Embedding Service.

---

## Module 5 — Embedding Pipeline

**Understand**: why embeddings exist, how embedding models work,
dimension sizes, distance metrics.

**Implement**: embedding generation, embedding storage, batch
processing, duplicate detection.

**Deliverables**: Embedding Module, Embedding Repository.

---

## Module 6 — Retriever

Implement semantic retrieval. Pipeline: Question → Embedding →
Similarity Search → Top K Documents → Ranking → Return Context.

**Deliverables**: Retriever, Similarity Search, Ranking Strategy,
Metadata Filtering.

---

## Module 7 — Prompt Builder

Construct prompts professionally. Prompt structure: System Prompt,
Retrieved Context, User Question, Safety Instructions, Expected
Format.

**Deliverables**: Prompt Templates, Context Injection, Citation
Strategy.

---

## Module 8 — LLM Integration

Integrate OpenAI or an open-source LLM.

**Responsibilities**: Prompt Generation, Response Generation,
Streaming, Token Usage, Retry Logic.

**Deliverables**: LLM Service, Configuration.

---

## Module 9 — Conversation Memory

Support: Conversation History, Previous Questions, Previous Answers,
Session Management, Context Window.

**Deliverables**: Conversation Repository, Memory Manager.

---

## Module 10 — Evaluation

Evaluate: Retrieval Accuracy, Precision, Recall, Latency, Hallucination
Rate, Groundedness, Answer Quality.

Create evaluation datasets.

**Deliverables**: Evaluation Reports, Metrics Dashboard.

---

## Module 11 — API Development

Build REST APIs, e.g. Upload Documents, Ask Question, List Documents,
Delete Documents, Search Documents, Conversation History.

**Deliverables**: FastAPI Backend, Swagger Documentation.

---

## Module 12 — Testing

Unit Tests, Integration Tests, Retriever Tests, Embedding Tests,
Prompt Tests, API Tests, Failure Tests.

Failure examples: Corrupted PDF, Missing Embedding, Empty Retrieval,
LLM Timeout, Database Failure.

---

## Module 13 — Performance Optimization

**Optimize**: Embedding Batch Size, Chunk Size, Chunk Overlap, Indexes,
Vector Search, Caching, Parallel Processing.

**Measure**: Latency, Memory, Throughput.

---

## Module 14 — Production Readiness

Authentication, Authorization, Logging, Monitoring, Docker,
Configuration Management, Secrets, Rate Limiting, Health Checks,
Backup Strategy, Disaster Recovery.
