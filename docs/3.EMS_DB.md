# Project 2: Education Management System (EMS)

I think we can improve it even further. Instead of just creating an
Education Management System, let's treat it like a real software
product that evolves over time — just as it would in a startup or
university.

## Technology Stack

- **Database:** PostgreSQL
- **Language:** Python
- **Driver:** `psycopg`
- **Architecture:** Repository Pattern + Service Layer
- **Goal:** Learn advanced PostgreSQL while building a production-style backend.

---

## Professional Project Goal

Build a backend for a university or online learning platform that supports:

- Student Management

- Enrollments

## Module 7 — Python Backend

Recommended structure:

```text
ems-backend/
├── database/
├── repositories/
├── services/
├── models/
├── scripts/
├── tests/
└── main.py
```

Responsibilities:

| Layer | Responsibility |
|---|---|
| `database` | Connection management |
| `repositories` | SQL only |
| `services` | Business logic |
| `models` | Entities |
| `scripts` | Schema creation, seed data, migrations |

## Module 9 — Advanced PostgreSQL

This is where PostgreSQL becomes the focus. Topics to implement naturally:

### CTEs

Build readable academic reports.

### Window Functions

- Student rank
- GPA ranking
- Department toppers

### Views

- Student dashboard
- Teacher dashboard
- Admin dashboard

### Materialized Views

- Semester reports
- Attendance summaries

### JSONB

Store flexible assignment settings, e.g. an `assignment_settings` field with:

- `submission_deadline`
- `rubric`
- `attachments`
- `late_penalty`

### Full Text Search

- Search courses
- Search teachers
- Search students

### Recursive Queries

- Department hierarchy
- Course prerequisites

## Learning Outcomes

After completing this project, you should be able to:

- Design normalized relational databases.
- Model complex one-to-one, one-to-many, and many-to-many relationships.
- Use PostgreSQL features such as CTEs, window functions, JSONB, views,
  materialized views, and `EXPLAIN ANALYZE`.
- Build a layered Python backend that separates database access from
  business logic.
- Write production-style reporting queries.
- Diagnose and optimize query performance.
- Explain your design decisions and trade-offs in technical interviews.
